aah framework

A scalable, performant, rapid development Web framework for Go

Get started with aah
Latest release v{{ config "aah_release.latest" }}, {{ config "aah_release.date" }}
Requires ≥ go1.8

aah framework makes it easy to build scalable web apps and mobile apps API with Go language. Framework goal is to achieve rapid, highly configurable, very small footprint and balanced Go web framework for small, medium, enterprise scale. Developer have to focus on application logic instead of building infrastructure, reinventing the wheel. aah values Don't Repeat Yourself and Convention Over Configuration, it makes application development more fun, free and open source.

Configuration

aah configuration is similar to Typesafe HOCON syntax. Flexible and easy to configure. aah is opinionated and defaults to set of conventions and configurations know more.

High Performance

Optimized Routing, HTTP/2, less GC overhead, optimized middlewares and pragmatic design.

Scalable

Build scalable web apps and mobile apps APIs comfortably. Framework is flexible, well structured and ability to fulfil modern application needs.

Security

The initial public release comes with Session Management. Goal is to address OWASP top 10 most critical web application security risks. Refer project roadmap for upcoming feature releases.

Rich Content Reply

aah supports various HTTP reply like HTML (mutli-layouts), JSON, JSONP, XML, Text, File (attachment, inline) and Bytes. Static file delivery via `http.ServeContent` and directory listing know more.

Developer Friendly

Development is more fun, typical method name(s) aligned with industry nomenclature, you have to just focus on your application logic, use aah CLI tool to qiuck start, cross-compile build, etc.
